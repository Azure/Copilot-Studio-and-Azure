# Solution Accelerators

Ready-to-deploy components that extend Copilot Studio with Azure AI capabilities. Each accelerator targets a specific integration pattern and can be deployed independently.

| Accelerator | What It Does | Why this accelerator? | Stack |
|---|---|---|---|
| [SharePoint Connector](sharepoint-connector/) | Lets employees ask a Copilot Studio agent natural-language questions about their SharePoint content and get grounded answers with citations — while only ever seeing documents they already have permission to read. Works for text-heavy and image-heavy content alike (policies, memos, slide decks, charts, scanned forms), keeps answers current as files are added, changed, or deleted, and can be scoped to a single site or folder so one team's content stays separate from the rest of the tenant. | Azure AI Search's preview SharePoint connector has no private-endpoint support, no Conditional Access compatibility, no SLA, and no per-user trimming. Building your own from scratch is typically a multi-month effort. This accelerator is a production-oriented starting point that already solves the three hardest parts — per-user security trimming, multimodal retrieval over text + images, and safe at-scale ingestion | Python, Azure Functions (Flex Consumption), Azure AI Vision multimodal + optional Document Intelligence Layout, Storage (Queue / Table / Blob), Key Vault (optional) |
| [Video RAG](#video-rag) | Processes video content for question-answering | Employee training libraries, recorded all-hands, product walkthroughs — information trapped in video is otherwise invisible to search. This accelerator turns video uploads into searchable, citeable knowledge without bespoke ML work. | Logic Apps, Content Understanding, Event Grid |
| [AI Search Flow](#ai-search-flow) | Search, index creation, and document upload from Power Platform | Gives low-code / citizen developers a supported Power Automate path into Azure AI Search — index creation, document upload, and semantic search — without writing Python or managing infrastructure. | Power Automate, Custom Connector |
| [Content Understanding Flow](#content-understanding-flow) | Document, image, audio, and video analysis from Power Platform | Same low-code pattern for multimodal analysis — pull structured data out of invoices, forms, screenshots, meeting recordings, etc., directly from a Power Automate flow. | Power Automate, Custom Connector |
| [Azure Copilot Pricing](#azure-copilot-pricing) | Real-time Azure pricing queries inside VS Code | Keeps Azure and Copilot Studio cost estimates accurate and current without leaving the editor — useful when scoping architectures or estimating credit consumption during design reviews. | GitHub Copilot Chat, REST API |
| [Voice Channel](voice-channel/) | Voice-enabled Copilot Studio agent without Omnichannel/Contact Center licensing. One Foundry "IT Assistant" agent serves three surfaces: custom Voice Live web UI, Microsoft Teams, and M365 Copilot Chat. Backend grounded via a Copilot Studio agent on the Microsoft Learn MCP server. | Voice real-time experience in Copilot Studio require additional licence. This accelerator demonstrates a supported path to voice-enabled Copilot Studio agents on the Foundry Agent Service side — one agent serving web, Teams, and M365 Copilot Chat | Azure AI Foundry (Voice Live, Agent Service), Container Apps, ACR, Copilot Studio, pac CLI |
